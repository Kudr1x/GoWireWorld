# GoWireWorld

Интерактивный симулятор клеточного автомата **Wireworld** на Go с графическим интерфейсом.

## Описание

**Wireworld** - это клеточный автомат, изобретённый Брайаном Сильверманом в 1987 году. Он предназначен для моделирования электронных схем и логических элементов. В отличие от Game of Life, Wireworld специально разработан для имитации передачи сигналов по проводам.

Этот симулятор позволяет:
- Рисовать схемы в реальном времени
- Запускать симуляцию с регулируемой скоростью
- Сохранять и загружать схемы
- Масштабировать и перемещать холст
- Работать с неограниченным полем

## Правила Wireworld

Каждая клетка может находиться в одном из четырёх состояний:

| Состояние | Цвет | Описание |
|-----------|------|----------|
| **Empty** (Пустая) | Чёрный | Пустое пространство |
| **Conductor** (Проводник) | Жёлтый | Проводящая среда |
| **Electron Head** (Голова электрона) | Синий | Передняя часть сигнала |
| **Electron Tail** (Хвост электрона) | Красный | Задняя часть сигнала |

### Правила перехода:

1. **Empty** -> остаётся **Empty**
2. **Electron Head** -> становится **Electron Tail**
3. **Electron Tail** -> становится **Conductor**
4. **Conductor** -> становится **Electron Head**, если рядом (в 8 соседних клетках) находятся **1 или 2** головы электронов, иначе остаётся **Conductor**

## Установка

### Требования

- Go 1.24 или выше
- Системные библиотеки для Ebitengine:
  - **Linux**: `libasound2-dev libgl1-mesa-dev libxcursor-dev libxi-dev libxinerama-dev libxrandr-dev libxxf86vm-dev`

### Установка зависимостей

#### Linux (Ubuntu/Debian)
```bash
sudo apt-get update
sudo apt-get install -y \
  libasound2-dev libgl1-mesa-dev libxcursor-dev libxi-dev \
  libxinerama-dev libxrandr-dev libxxf86vm-dev
```

### Сборка

```bash
git clone https://github.com/Kudr1x/GoWireWorld.git
cd GoWireWorld
go build -o gowireworld ./src/
```

## Использование

### Запуск

```bash
./gowireworld
```

### Управление

#### Мышь
- **ЛКМ** - рисовать выбранным состоянием
- **ЛКМ + перетаскивание** - рисовать линию
- **СКМ** - перемещать холст
- **Колесо мыши** - масштабирование (от 4x до 64x)

#### Панель управления (слева)

**Кнопки состояний:**
- Чёрная - выбрать Empty (стирание)
- Жёлтая - выбрать Conductor (проводник)
- Синяя - выбрать Electron Head (голова)
- Красная - выбрать Electron Tail (хвост)

**Кнопки действий:**
- **Start/Stop** - запустить/остановить симуляцию
- **Ползунок скорости** - от 1 до 10 итераций/секунду
- **Save** - сохранить схему в файл (.wws)
- **Load** - загрузить схему из файла

### Примеры схем

В папке `schemes/` находятся готовые схемы:
- `baseGate.wws` - базовые логические элементы (NOT, OR, AND, NAND, XOR)
- `binsum.wws` - двоичный сумматор

Загрузите их через кнопку **Load** в интерфейсе.

## Архитектура

Проект разделён на два независимых пакета для чистой архитектуры и быстрого тестирования:

```
src/
├── core/              # Бизнес-логика
│   ├── types.go       # Определения типов
│   ├── logic.go       # Алгоритм Wireworld
│   └── logic_test.go  # Unit-тесты
│
├── game/              # UI слой (Ebiten)
│   ├── types.go       # Структура Game
│   ├── constants.go   # UI константы и цвета
│   ├── render.go      # Отрисовка сцены
│   ├── update.go      # Обработка событий
│   ├── utils.go       # Вспомогательные функции
│   └── io.go          # Сохранение/загрузка
│
└── main.go            # Точка входа
```

## Технические детали

### Оптимизация производительности

- **Конкурентные вычисления** - симуляция запускается в отдельной горутине
- **Mutex для безопасности** - защита от race conditions при доступе к полю
- **Ленивая инициализация** - текстуры создаются только при первом использовании
- **Viewport culling** - отрисовываются только видимые клетки

### Формат файла .wws

Простой текстовый формат:
```
<x> <y> <state>
<x> <y> <state>
...
```

Пример:
```
0 0 1
1 0 1
2 0 2
```

Где `state`: `1` = Conductor, `2` = Electron Head, `3` = Electron Tail
